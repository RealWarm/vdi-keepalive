# VDI Keep-Alive

Citrix Workspace 등 VDI 창이 무활동 타임아웃으로 끊기는 것을 막기 위한 AutoHotkey v2 스크립트입니다.

설정한 주기마다 대상 창을 잠깐 포커스 → 마우스 1픽셀 이동 → 원위치 → 이전 활성 창으로 복귀 합니다.
화면에 잠깐 깜빡임만 있을 뿐, 사용자 작업 흐름은 거의 그대로 유지됩니다.

## 설치 / 실행 순서

1. **AutoHotkey v2 설치**
   - https://www.autohotkey.com 에서 "Download v2.0" 다운로드 후 설치
2. 이 폴더(`vdi-keepalive/`)를 Windows 측 원하는 위치로 복사
   - 예: `C:\Users\<본인 계정>\Tools\vdi-keepalive`
3. **`config.ini` 수정**
   - `[Targets]` 의 `Keyword1=` 값을 본인 Citrix 창 제목에 들어있는 단어로 설정
   - 보통 `Citrix` 한 단어로 충분 (창 제목이 `... - Citrix Workspace` 같은 형태이므로)
   - 다른 종류의 창도 같이 살리고 싶다면 `Keyword2=`, `Keyword3=` 에 추가
4. `vdi-keepalive.ahk` **더블클릭**으로 실행
5. 작업 표시줄 오른쪽 **시스템 트레이에 H 아이콘**이 생기면 정상

## 트레이 메뉴 (아이콘 우클릭)

- **일시정지 / 재개** — 토글
- **지금 한 번 실행** — 즉시 동작 (테스트용)
- **설정 파일 열기** — `config.ini` 메모장으로 열기 (수정 후 스크립트 재시작 필요)
- **종료**

## 동작 확인 방법

1. Citrix Workspace를 띄워 둡니다 (창 제목에 `Citrix` 포함되는지 확인)
2. 메모장이나 브라우저 같은 다른 프로그램을 활성화 상태로 둡니다
3. 트레이 아이콘 우클릭 → **"지금 한 번 실행"** 클릭
4. 화면이 **잠깐 Citrix로 전환됐다가 원래 프로그램으로 복귀**하면 정상
5. 마우스 커서가 원래 자리에 그대로 있는지 확인
6. 그 상태로 5분 대기 → 자동으로 같은 동작이 일어나는지 확인

## 설정 파일 (`config.ini`)

| 항목 | 의미 | 기본값 |
|---|---|---|
| `IntervalSec` | 동작 주기 (초) | `300` (5분) |
| `WindowSwitchPauseMs` | 포커스 전환 후 마우스 이동까지 대기 (ms) | `80` |
| `SkipIfMinimized` | 최소화된 창은 건드리지 않음 (1=on, 0=off) | `1` |
| `Keyword1`, `Keyword2`, ... | 대상 창 제목 키워드 (부분일치, 대소문자 무시) | `Citrix` |

설정 변경 후에는 **트레이 아이콘에서 종료 → 다시 스크립트 더블클릭** 으로 재시작하세요.

## Windows 시작 시 자동 실행 (선택)

1. `Win + R` 누르고 `shell:startup` 입력 후 엔터 (시작프로그램 폴더가 열림)
2. `vdi-keepalive.ahk` 를 오른쪽 클릭 → **"바로 가기 만들기"**
3. 만들어진 바로 가기를 시작프로그램 폴더로 이동

## 주의 사항

- 동작 순간에는 **Citrix 창이 잠깐 활성화** 됩니다. 그 짧은 순간(약 80ms) 키보드 입력을 하고 있으면 입력이 Citrix로 들어갈 수 있습니다. 5분 간격이라 실제 충돌 가능성은 낮지만 유의하세요.
- VDI 자체 세션 정책(예: 서버에서 강제 절단)을 우회하지는 못합니다. 어디까지나 **클라이언트 측 무활동 감지를 회피**하는 용도입니다.
- Citrix 창이 최소화 상태이면 기본 설정상 건드리지 않습니다 (`SkipIfMinimized=1`). 최소화 상태에서도 살리고 싶으면 `0` 으로 바꾸세요. 단, 그 경우 창이 복원되며 화면에 튀어나옵니다.

## 문제 해결

- **트레이에 메시지박스가 뜨고 종료됨**: `config.ini` 의 `Keyword1=` 이 비어 있습니다. 값을 채워 넣으세요.
- **5분이 지나도 아무 일도 일어나지 않음**:
  - 트레이 메뉴에서 "지금 한 번 실행" 을 눌러봐서 동작 자체가 되는지 확인
  - 동작 안 되면 `Keyword1` 이 실제 Citrix 창 제목에 포함되어 있는지 확인 (창 제목 표시줄 또는 작업 표시줄 미리보기로 확인)
- **"지금 한 번 실행" 해도 깜빡임이 없음**: 대상 창이 최소화 상태일 가능성. `SkipIfMinimized=0` 으로 바꾸고 재시작.
- **AutoHotkey v1만 설치되어 있어 실행이 이상함**: AutoHotkey v2를 추가 설치하세요. v1/v2는 공존 가능합니다.
